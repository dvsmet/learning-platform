# Корпоративная платформа обучения (Learning Platform)

Полнофункциональное веб-приложение для внутреннего обучения сотрудников: каталог курсов, уроки, тестирование, прогресс, чат с инструктором, отзывы и развитый модуль аналитики с визуализацией и выгрузкой отчётов.

## Кратко

**Learning Platform** — это корпоративный LMS-портал (Learning Management System) с разделением на три роли (сотрудник, инструктор, администратор). Проект реализует полный цикл обучения: от заявки на курс до сдачи тестов и аналитики результатов.

Архитектура построена как единый сервис: **ASP.NET Core Web API** на бэкенде + **React SPA** на фронтенде, собираемый в статику и раздаваемый тем же сервером. Данные хранятся в **PostgreSQL**, доступ через **Entity Framework Core** с миграциями.

## Ключевые возможности

### Сотрудник
- Каталог курсов с поиском и фильтрами
- Подача заявки на доступ к курсу
- Прохождение уроков с отслеживанием прогресса
- Сдача тестов с автоматической проверкой и подсчётом баллов
- Личный прогресс по каждому курсу
- Чат с инструктором, отзывы на курсы

### Инструктор
- Управление своими курсами, уроками и тестами
- Обработка заявок на доступ
- Чат с обучающимися своих курсов
- Аналитика по своим курсам и студентам

### Администратор
- Управление пользователями (роли, сброс паролей)
- Полное управление контентом (курсы, категории, уроки, тесты)
- Модерация отзывов и заявок
- **Полная аналитика обучения**: KPI, таблицы, детализация по сотрудникам
- **Круговая диаграмма успеваемости** с фильтром по датам и экспортом в PNG
- **Выгрузка отчётов в Excel** с выбором периода

## Основной бизнес-процесс

1. Сотрудник оформляет **заявку (Order)** на курс
2. Инструктор/админ **одобряет** заявку (`status = approved`)
3. `CourseAccessService` автоматически создаёт записи прогресса по урокам — открывается доступ
4. Сотрудник проходит **уроки** и **тесты**, результаты сохраняются
5. Доступен **чат** с инструктором по курсу
6. После прохождения сотрудник оставляет **отзыв**

Все этапы отражаются в **аналитике** для инструктора и администратора.

## Модуль аналитики

Один из ключевых и наиболее проработанных модулей проекта.

- **Дашборд**: KPI по курсам (записи, средний прогресс, % завершения, средний балл тестов)
- **Обучающиеся**: таблица с сортировкой и детальной карточкой (прогресс, статусы тестов)
- **Распределение баллов** по тестам каждого курса (гистограммы)
- **Сводка по инструкторам** (для администратора)
- **Круговая диаграмма успеваемости**: «не приступили» / «сдали (≥60%)» / «не сдали или в процессе» с фильтром по дате заявки и сохранением в PNG
- **Экспорт в Excel** (ClosedXML): многостраничный отчёт с выбором периода (неделя / месяц / год / произвольные даты)

## Запуск проекта

### Требования
- .NET SDK 8.0+
- Node.js 18+
- PostgreSQL

### Настройка БД

Строка подключения задаётся в `appsettings.json` (ключ `ConnectionStrings:DefaultConnection`). Миграции применяются автоматически при старте приложения.

### Сборка и запуск

```powershell
# 1. Сборка фронтенда
cd client-app
npm install
npm run build
cd ..

# 2. Сборка и запуск бэкенда
dotnet restore
dotnet run --project MyWebApi.csproj
```

### Режим разработки фронтенда

```powershell
cd client-app
npm run dev
```

### Публикация

```powershell
npm run build --prefix client-app
dotnet publish MyWebApi.csproj -c Release -o publish
```

## Аутентификация и роли

- Вход по email и паролю; пароли хранятся в виде **BCrypt-хешей**
- Идентификатор пользователя передаётся в заголовке `X-User-Id`
- Маршрутизация в SPA защищена по ролям (`ProtectedRoute`):
  - сотрудник → `/dashboard`
  - инструктор → `/instructor`
  - администратор → `/admin`
